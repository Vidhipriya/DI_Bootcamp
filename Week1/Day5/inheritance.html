Children tags inherit the properties and values of their parents when it is not specified. When it is specified individually, it is similar to breaking that much needed lazy cycle.

Combinator selectors

They work in regards to inheritance. Let's take the example of
div p {}

The space observed between tags in this combinator selector is like the generation gaps between you and your ancestors but your eyes are still a beautiful brown because like p you have inherited the characterics of your ancestor. The example illustrated above is called the descendant selector. Next up we will investigate the example of
article > p {}

The greater than symbol(>) can be seen as parents introducing their children to others and them saying proudly "yes this is my child right here" while pointing at them. And you guessed it right ,this combinator selector is called a child selector. Another illustration that is as interesting is
h1 + p {}

The addition symbol is used for adjacent sibling selectors in the same way it's always a plus to have them around and help with the chores...but also in your life ofcourse.

As we have seen previously, pseudo-class selectors bring out a specific state of a tag. We can see that in terms of how sometimes genes can skip generations ony to be applied to someone.

# Pseudo-class Selector Usage
1 :empty Represents an element with no children other than white-space characters.
2 :nth-child Uses An+B notation to select elements from a list of sibling elements. Uses An+B notation to select elements from a list of sibling elements.
3 :nth-last-child Uses An+B notation to select elements from a list of sibling elements, counting backwards from the end of the list.d
3 :first-child Matches an element that is the first of its siblings.
3 :last-child Matches an element that is the last of its siblings.
3 :only-child Matches an element that has no siblings. For example a list item with no other list items in that list
3 :nth-of-type Uses An+B notation to select elements from a list of sibling elements that match a certain type from a list of sibling elements.
3 :first-of-type Matches an element that is the first of its siblings, and also matches a certain type selector
3 :last-of-type Matches an element that is the last of its siblings, and also matches a certain type selector.
3 :only-of-type Matches an element that has no siblings of the chosen type selector.
